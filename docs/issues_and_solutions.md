1. Mahalanobis OOD ill-conditioning. Reduce embeddings to a 64-dim PCA subspace fit on the training set, then compute Mahalanobis there. PCA basis, mean, and covariance inverse are cached alongside the embedding manifest, keyed by the same backbone × preprocessing hash. Alternative if PCA is undesirable: Ledoit-Wolf shrinkage on the full 1024-dim covariance via sklearn.covariance.LedoitWolf. Default to PCA-64; Ledoit-Wolf as a fallback if PCA loses too much signal. Threshold calibration (95th percentile) happens in the reduced space, not the original.
2. Multi-source batch imbalance. Equal-sized per-source mini-batches via a custom BalancedSourceSampler that draws N/k samples from each of k sources per batch (k = number of active datasets). Smaller datasets get oversampled within an epoch; larger ones get subsampled. Configurable strategy field in the training config: source_balancing: {proportional, equal, sqrt}. Default to equal when multi-source training is on; default falls away when only one dataset is active. The auxiliary source-classifier head still does its invariance job on top.
3. Reliability lookup needs its own split. Move from 70/15/15 (train/val/test) to 60/15/15/10 (train/val/reliability/test). The reliability split is never seen during hyperparameter tuning, never feeds into early stopping, and is held out until after final model selection. The dashboard's "AUC for patients like you" table is computed on this split. Same patient-grouping discipline. The split utility takes a list of ratios and split names, so this is one config change, not a code change.
4. Retinal age on ODIR. Treat retinal age as a derived dashboard convenience under the ODIR-only scenario, not a headline output. Two changes flow from this: the task registry tags retinal age with headline_status: secondary for ODIR but headline_status: primary for BRSET; the paper's results section reports retinal age MAE as a secondary number in a "model-as-biomarker" subsection rather than in the headline table. If BRSET arrives, this auto-upgrades. The retinal-age-gap-as-biomarker claim is reserved for BRSET runs only.
5. OOD threshold under camera-class shift. Default to single global threshold calibrated on the source-distribution validation split, with the high mBRSET rejection rate framed as the protocol working as intended (gating prevents the model from adapting to inputs it shouldn't). One labelled ablation runs per-stream-class thresholds (one for clinical-camera streams, one for smartphone streams) and reports the difference. The paper's main continual-learning table uses the global threshold; the ablation table shows what changes under per-class thresholds. This gives reviewers both views without making the headline number depend on a tuning choice.
6. TCAV concept exemplars. Drop hand-curated clinical concepts. Use RFMiD as the concept-direction source: its 45-49 multi-label vocabulary already names retinal findings (drusen, hemorrhages, optic disc edema, etc.) and the labelled examples become the concept exemplars directly. Concept directions computed by training a linear classifier on the foundation-model embeddings of RFMiD-labelled vs. unlabelled examples for each concept; the classifier's normal vector is the concept direction. This is methodologically the same as TCAV; the only change is the source of exemplars. If RFMiD's label vocabulary doesn't cover a concept the paper wants to discuss, that concept is omitted rather than improvised.
7. RFMiD's role. Pin it as concept-direction source for TCAV (per #6) plus secondary external validation for multi-label ocular breadth. Not a continual-learning stream component (its label schema differs from ODIR's eight categories enough that streaming it is messy). Two roles, both well-defined: the configs/dataset/rfmid.yaml file declares roles: [tcav_concepts, external_eval_ocular], and the dataset role matrix in the spec is updated.
8. ConvNet baseline embedding dim. Skip cross-attention for the ConvNet baseline. The fairness comparison is "ViT foundation models with cross-attention vs. ConvNet ImageNet baseline without cross-attention," which is the realistic deployment comparison anyway (you wouldn't graft cross-attention onto an ImageNet ResNet for a real screening system). The fusion trunk's first-layer input dim is config-driven via backbone.embedding_dim, so the ResNet-50's 2048 or ConvNeXt-Base's 1024 plug in directly. The cross-attention block is gated by model.use_paired_eye_attention per the existing config; for the ConvNet run, this is set to false. Reported as an architectural difference in the methods section, not as a confound.
